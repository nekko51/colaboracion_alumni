# File formatting info:

## print_metropolis_data_to_file;
The output file generated by `run_metropolis`'s call to `print_metropolis_data_to_file` is a report summarizing the simulation results, which consists of:

1.  **Header**: A line indicating the report version.
    ```
    ***************Metropolis data report version 1.0***************
    ```

2.  **Global Parameters**: Parameters of the entire simulation run.
    *   `weight_log`, `weight_properties`: The weights used for energy calculation.
    *   `n_betas`: The total number of beta values simulated.
    *   `n_steps`: The number of Metropolis sweeps performed for each beta.

3.  **Overall Statistics**:
    *   `overall acceptance med`: The mean of the mean acceptance rates from each beta simulation.
    *   `overall acceptance var`: The variance of the mean acceptance rates.

4.  **Initial State**: Information about the starting sequence before any simulation steps.
    *   `log humanness`, `prop humanness`, `total energy`: Energy components of the initial sequence.
    *   `seed`: The initial murine sequence seed.

5.  **Per-Beta Results**: This section is repeated for each beta value.
    *   A header line showing the current `beta` value, its index, the mean acceptance rate for that beta (`acceptance med`), and the variance of the acceptance rate (`acceptance var`).
    *   `delta_e`: The change in total energy between the result of the previous beta and the current one.
    *   `log humanness`, `prop humanness`, `total energy`: Energy components of the final sequence for this beta.
    *   `resulting chain`: The final, (hopefully) more humane sequence after all sweeps at this beta are complete.

6.  **Footer**: A line marking the end of the report.
    ```
    ***************Metropolis data report end***************
    ```
